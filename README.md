# Java exam management platform
I implemented the following functionalities:
- Main class: where I read and parsed the commands, which implements each functionality in Platform.java;
- Platform class: depending on each command, performs the necessary operations (for example, create_exam, add_question etc.);
- Question class: is an abstract class that implements Gradable and contains: specific attributes, getters and setters, the abstract method checkAnswer() which is later implemented in the classes that inherit the current class (the type of data received is implemented using generics). The parameter that the method receives is of type String or ResponseOption, depending on the type of question.
- OpenEndedQuestion, FillInTheBlankQuestion and MultipleChoiceQuestion classes: are the classes that inherit the Question class and represent the 3 types of questions that must be handled differently. Thus, the classes implement the checkAnswer() method from the parent class and the method for calculating the percentage of the score from the Gradable interface (getGrade()), checking whether an answer is correct, partially correct or incorrect, depending on the conditions established by each type of question;
- Exam class: implements functionalities for operations related to Exam type objects. For example, addIntrebare() adds a new question to the list, sortIntrebari() and sortIntrebari2() sort the list in the 2 required ways (chronological + author, respectively difficulty + alphabetical), sortScor() sorts the students according to the obtained score, submitExam() calculates the total score of the student (the method also being used in Platform) and submitBoundsCheck() checks whether the student submitted the exam in the correct interval;
For history, the first sorting method performs the sorting by the date of addition, then, in case of equality by author, and the second sorting method performs the sorting by difficulty, then by the text of the question.
- Student class: contains a Map collection for examName and score, in order to efficiently model the exams that the student has taken and the score for each of them;
- SubmissionOutsideTimeIntervalException exception: is thrown when the student did not submit in the correct interval (between the start date and the end date of the exam). The exception is thrown in the submitExam() method from Exam if the timestamp of the submission is outside the correct interval, the student's score being set to 0.0. We write the exception message in console.out in the format `<timestamp> | Submission outside of time interval for student <studentName>`;
- Gradable interface: contains the abstract method getGrade() which is later implemented in the classes that inherit Question;
- Correctness and ResponseOption enums: contain the degree of correctness of an answer, respectively the possible answers within a multiple-choice question;
Input/Output: the commands are read from the file at the path `src/main/resources/<testName>/input.in`, where the name of the test is the argument from main, and the results are written in the .out files from the same folder of the test. After the writing is finished, we close() the filewriter, respectively the consolewriter.
I used the following collections:
- in Platform I used 2 hashmaps: one to find an exam based on its name and one to find a student based on their name;
- in Exam I used an arraylist for questions, adding them and sorting, and a hashmap to perform the mapping between the student's name and their score;
- in Student I used a hashmap between the name of the exam and the score obtained at that exam.
I used the hashmaps because they have good complexity for get and put operations (O(1)), being an efficient method to keep the connection between the component elements. The arraylist also has O(1) complexity for the add operation, and for sorting I used Collections.sort with O(nlogn) complexity.
The hashmaps help us within the Platform class, where operations of identifying an exam or a student based on name are frequently necessary, and the arraylist is useful for keeping the questions, the answers at submit being in the order of the printed questions.
The collections used in the implementation are efficient also for interleaved operations, in the statement being mentioned the fact that, for example, the add_question command can be given several times consecutively, followed by print_exam, after which questions can continue to be added and so on.
